# Documentation map

## Public technical documentation

- `API_INTEGRATION.md`: authenticated and legacy ANA access model.
- `DATABASE_SCHEMA.md`: stable publication-database contracts.
- `DATA_SOURCES_AND_LIMITATIONS.md`: source, scope and interpretation limits.
- `DEPENDENCIES.md`: runtime, pipeline and system dependency notes.
- `DIAGNOSTIC_METHODS.md`: public description of screening methods.
- `FUTURE_ARCHITECTURE.md`: deferred major-version backend proposal.
- `DEPLOYMENT.md`: repository preparation and staged deployment workflow.
- `RELEASE_CHECKLIST.md`: public regression checklist.
- `publication/`: publication-batch and rollback records.

Internal chat history, local audit workspaces, private paths and full development decision logs are intentionally retained only in the private baseline and are not part of the public repository.
